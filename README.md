## Overview

This repository documents the completion of the Scanpy basic scRNA-seq tutorial using the canonical **PBMC3k dataset** — 2,700 peripheral blood mononuclear cells sequenced on the 10x Genomics Chromium platform. The analysis covers the full single-cell RNA-seq downstream analysis workflow: from raw count matrix to annotated cell type clusters.

Single-cell RNA sequencing (scRNA-seq) allows the measurement of gene expression in individual cells rather than bulk tissue, revealing cellular heterogeneity that would otherwise be masked. This tutorial demonstrates how to process, cluster, and annotate single-cell data using the Scanpy ecosystem.

## Pipeline Summary

```
Raw count matrix (10x MTX format)
         │
         ▼
  [AnnData object]
  Load into .h5ad format
         │
         ▼
  [Quality Control]
  Filter low-quality cells & genes
  MT gene % · n_genes · n_counts
         │
         ▼
  [Normalisation + Log transform]
  Total-count normalise → log1p
  Freeze raw counts
         │
         ▼
  [Highly Variable Genes]
  Select top 2,000 HVGs
         │
         ▼
  [Regress out confounders]
  n_counts · percent_mito
         │
         ▼
  [PCA]
  50 principal components
         │
         ▼
  [Neighbourhood graph]
  k-NN graph (k=10, n_pcs=40)
         │
         ▼
  [UMAP + t-SNE]
  2D embeddings for visualisation
         │
         ▼
  [Louvain clustering]
  Community detection on k-NN graph
         │
         ▼
  [Marker genes]
  rank_genes_groups (Wilcoxon)
         │
         ▼
  [Cell type annotation]
  Cluster labels → known cell types
```

## Step-by-Step Workflow

### 1. Data Loading

The raw count matrix was read from the 10x MTX format into an **AnnData** object using Scanpy. AnnData is the standard data structure for single-cell analysis in the scverse ecosystem — it stores the expression matrix alongside cell and gene metadata.

```python
import scanpy as sc

sc.settings.verbosity = 3
sc.set_figure_params(dpi=80, facecolor="white")

results_file = "results/pbmc3k.h5ad"

adata = sc.read_10x_mtx(
    "data/filtered_gene_bc_matrices/hg19/",
    var_names="gene_symbols",
    cache=True,
)
adata.var_names_make_unique()
```

**AnnData structure:**

| Slot | Contents |
|---|---|
| `adata.X` | Count matrix (cells × genes) |
| `adata.obs` | Cell metadata (QC metrics, cluster labels) |
| `adata.var` | Gene metadata (HVG flags, means, dispersions) |
| `adata.uns` | Unstructured data (PCA, neighbours, colours) |
| `adata.obsm` | Embeddings (PCA, UMAP, t-SNE coordinates) |

**Initial object:** 2,700 cells × 32,738 genes

---

### 2. Quality Control

Low-quality cells were identified and removed based on three metrics:

| Metric | Threshold | Rationale |
|---|---|---|
| `n_genes_by_counts` | < 200 | Cells with too few genes may be empty droplets |
| `n_genes_by_counts` | > 2,500 | Cells with too many genes may be doublets |
| `pct_counts_mt` | > 5% | High mitochondrial content indicates dying/damaged cells |

Mitochondrial genes were flagged using the `MT-` gene name prefix:

```python
adata.var["mt"] = adata.var_names.str.startswith("MT-")
sc.pp.calculate_qc_metrics(
    adata, qc_vars=["mt"], percent_top=None, log1p=False, inplace=True
)

# Apply filters
sc.pp.filter_cells(adata, min_genes=200)
sc.pp.filter_genes(adata, min_cells=3)
adata = adata[adata.obs.n_genes_by_counts < 2500, :]
adata = adata[adata.obs.pct_counts_mt < 5, :]
```

**After QC:** 2,638 cells × ~13,714 genes  
(57 cells removed for high MT%, ~18 cells removed as potential doublets)

---

### 3. Normalisation

Total-count normalisation was applied to remove differences in sequencing depth between cells, followed by log transformation to stabilise variance:

```python
# Normalise each cell to 10,000 total counts
sc.pp.normalize_total(adata, target_sum=1e4)

# Log-transform: log(x + 1)
sc.pp.log1p(adata)

# Freeze normalised counts as the "raw" layer for later use
adata.raw = adata
```

> **Why freeze raw?** The raw log-normalised values are stored before further transformations (scaling, regressing out). This preserves full gene expression data for downstream differential expression testing, which should not be run on scaled data.

---

### 4. Feature Selection

Highly variable genes (HVGs) were identified to reduce dimensionality and focus downstream analysis on biologically informative genes:

```python
sc.pp.highly_variable_genes(
    adata,
    layer="counts",
    n_top_genes=2000,
    min_mean=0.0125,
    max_mean=3,
    min_disp=0.5,
    flavor="seurat_v3",
)

# Retain only HVGs
adata = adata[:, adata.var.highly_variable]
```

**Result:** 2,000 highly variable genes selected from ~13,714

Confounder effects (total counts and mitochondrial percentage) were then regressed out, and gene expression was scaled to unit variance (values clipped at 10 standard deviations):

```python
sc.pp.regress_out(adata, ["total_counts", "pct_counts_mt"])
sc.pp.scale(adata, max_value=10)
```

---

### 5. Dimensionality Reduction

#### PCA

PCA was applied to the scaled HVG matrix to reduce dimensionality while capturing the major axes of variation:

```python
sc.tl.pca(adata, svd_solver="arpack")
sc.pl.pca_variance_ratio(adata, log=True, n_pcs=50)
```

The variance ratio plot was used to select the number of PCs to use downstream (40 PCs captured the majority of variance without including noise-dominated components).

#### Neighbourhood Graph

A k-nearest neighbour (k-NN) graph was constructed in PCA space:

```python
sc.pp.neighbors(adata, n_neighbors=10, n_pcs=40)
```

This graph is the foundation for both clustering (Louvain) and UMAP embedding — cells that are transcriptionally similar are connected by edges.

#### UMAP and t-SNE

Two-dimensional embeddings were computed for visualisation:

```python
sc.tl.umap(adata)
sc.tl.tsne(adata)
```

---

### 6. Clustering

The Louvain community detection algorithm was applied to the k-NN graph to identify cell clusters:

```python
sc.tl.louvain(adata)
```

Louvain clustering identifies groups of cells with high internal connectivity (transcriptionally similar cells) at a resolution that is automatically determined by the graph structure. The result is stored in `adata.obs["louvain"]`.

**Result:** 8 distinct clusters identified

---

### 7. Marker Gene Identification

For each cluster, marker genes were identified using the Wilcoxon rank-sum test comparing each cluster against all others:

```python
sc.tl.rank_genes_groups(adata, "louvain", method="wilcoxon")
sc.pl.rank_genes_groups(adata, n_genes=25, sharey=False)
```

Top marker genes were examined per cluster to guide cell type annotation.

---

### 8. Cell Type Annotation

Clusters were manually annotated based on the expression of known marker genes for PBMC cell types:

| Cluster | Cell type | Key marker genes |
|---|---|---|
| 0 | CD4+ T cells | IL7R, CCR7 |
| 1 | CD14+ Monocytes | CD14, LYZ |
| 2 | CD4+ T cells (memory) | IL7R, S100A4 |
| 3 | B cells | MS4A1, CD79A |
| 4 | CD8+ T cells | CD8A, CD8B |
| 5 | NK cells | GNLY, NKG7 |
| 6 | FCGR3A+ Monocytes | FCGR3A, MS4A7 |
| 7 | Dendritic cells / Platelets | FCER1A, CST3 |

```python
cell_type_map = {
    "0": "CD4+ T cells",
    "1": "CD14+ Monocytes",
    "2": "CD4+ T cells (memory)",
    "3": "B cells",
    "4": "CD8+ T cells",
    "5": "NK cells",
    "6": "FCGR3A+ Monocytes",
    "7": "DC/Platelets",
}
adata.obs["cell_type"] = adata.obs["louvain"].map(cell_type_map)
```

## Discussion

The PBMC3k dataset is a well-characterised benchmark dataset, and the results obtained here closely match the expected biology of peripheral blood. Eight distinct cell populations were recovered, corresponding to the major PBMC subtypes: T cells (CD4+ naive and memory, CD8+), B cells, NK cells, monocytes (classical CD14+ and non-classical FCGR3A+), and a small cluster of dendritic cells and platelets.

The separation of T cell subtypes (naive CD4+ vs memory CD4+ vs CD8+) demonstrates the resolution achievable with scRNA-seq — these populations cannot be distinguished by bulk RNA-seq. The clean separation of the two monocyte populations (CD14+ and FCGR3A+) in UMAP space further illustrates how dimensionality reduction captures biologically meaningful variation.

The UMAP embedding produced more interpretable cluster separation than t-SNE, which is consistent with observations in the broader scRNA-seq community — UMAP better preserves global structure while maintaining local relationships.

A key limitation of this analysis is that cell type annotation was performed manually based on known marker genes. This approach works well for well-characterised cell types in a standard tissue (peripheral blood), but would require more careful consideration for novel cell populations or less-studied tissues.
